ANIVA Sentry 設定ガイド

現在の状態

  • ✅ Sentry SDK インストール済み（@sentry/nextjs v10.42.0）
  • ✅ DSN 設定済み（.env + .env.staging）
  • ✅ Client/Server/Edge 初期化完了
  • ✅ next.config.ts に withSentryConfig 統合済み
  • ✅ PII除外（パスワード・トークン自動フィルタ）

ダッシュボードへのアクセス

  1. https://k-rascal.sentry.io にアクセス
  2. Keisukeのアカウントでログイン
  3. プロジェクト「javascript-nextjs」を選択

見るべき画面

  • Issues: 発生しているエラー一覧。頻度・影響ユーザー数でソート
  • Performance: ページ/APIのレスポンスタイム。遅いルートを特定
  • Alerts: エラー急増時の通知設定（Slack/メール）

おすすめ設定

  1. Slack通知: Settings → Integrations → Slack → #aniva-product に新規エラーを通知
  2. アラートルール: Alerts → Create Alert → 「1時間に同じエラー10回以上で通知」
  3. リリース追跡: 自動（withSentryConfigが各デプロイをリリースとして記録）

設定値

NEXT_PUBLIC_SENTRY_DSN=https://a053...@o451...ingest.us.sentry.io/451...
SENTRY_ORG=k-rascal
SENTRY_PROJECT=javascript-nextjs
tracesSampleRate=0.1（10%サンプリング）
replaysOnErrorSampleRate=1.0（エラー時100%リプレイ）
